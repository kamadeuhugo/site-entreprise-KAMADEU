## Stratégie de branchement
- master : branche finale stable
- dev : intégration globale avant publication
- test/frontend : tests d'intégration intermédiaires
- feature/html-structure : structure HTML des pages
- feature/css-theme : charte graphique CSS responsive
- feature/js-interactions : interactions JS dynamiques

## Flux de travail
feature/* → test/frontend → dev → master

## Etapes suivies
1. Initialisation du dépôt Git
2. Création de toutes les branches
3. Développement HTML sur feature/html-structure
4. Développement CSS sur feature/css-theme
5. Développement JS sur feature/js-interactions
6. Fusion des features dans test/frontend
7. Fusion test/frontend dans dev
8. Fusion dev dans master

## Commandes Git utilisées
- git init / git clone
- git branch / git checkout / git switch
- git add / git commit
- git merge
- git log / git status
- git push / git pull
